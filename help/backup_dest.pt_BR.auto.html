Destino de backup
Esses campos controlam para onde o backup do Virtualmin é gravado. Você pode escolher entre um dos três modos de destino:
Diretório de arquivo local
Nesse modo, o backup é simplesmente salvo em um arquivo no sistema executando o Virtualmin. Se você estiver usando o formato de backup de arquivo único , deverá inserir um nome de arquivo aqui, como /backup/domainname.tgz . Se estiver usando um dos formatos Um arquivo por servidor , você precisará inserir um diretório, como /backup .
servidor FTP
Este modo pode ser usado para carregar o arquivo ou arquivos de backup usando o protocolo FTP para outro servidor. Você deve inserir o nome do servidor, login, senha e arquivo ou diretório de destino.
Servidor SSH
Isso é semelhante ao modo FTP, mas transfere o backup usando o protocolo SSH.
Se o formato de backup for um único arquivo, o destino que você especificar aqui deverá ser um arquivo que será substituído ou criado. Se você criar um arquivo por domínio, o destino deverá ser um diretório que já exista.

Ao fazer backup em um servidor SSH ou FTP em uma porta não padrão, você pode inseri-lo no campo do servidor separado do nome do host com dois pontos, como backupserver.yourdomain.com:2222 .

Se estiver fazendo backup em um servidor SSH que suporte apenas autenticação baseada em certificado, o campo de senha é opcional. No entanto, o usuário root no sistema deve ter sua chave pública SSH no arquivo authorized_keys do usuário de destino.
