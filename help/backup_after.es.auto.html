Comando para ejecutar después de la copia de seguridad
Si se establece, el comando o script ingresado se ejecutará como root después de que la copia de seguridad se ejecute según lo programado. Esto puede ser útil para desmontar sistemas de archivos remotos o extraíbles en los que se escribió la copia de seguridad, o para limpiar archivos de copia de seguridad antiguos.
