Sekundäre Mailserver
Ein sekundärer Mailserver kann auch E-Mails für einen der von Virtualmin verwalteten virtuellen Server empfangen, jedoch nur als temporäres Relay und nicht als endgültiges Ziel. Es ist ratsam, einen sekundären Server für häufig verwendete Mail-Domänen einzurichten, der als Sicherungs- oder Aufbewahrungsort für den Fall dient, dass der primäre Mail-Server ausfällt.

Auf dieser Seite können Sie andere Server auswählen, auf denen Virtualmin installiert ist, um als sekundäre MX-Server für neue Mail-Domänen verwendet zu werden, die auf diesem System erstellt wurden. Bevor Server angezeigt werden, müssen sie zuerst dem Webmin Servers Index-Modul hinzugefügt werden, in dem Sie das Root-Login und das Kennwort für den Remote-Server eingeben müssen.

Für jeden sekundären Mailserver haben Sie die Möglichkeit, anstelle des tatsächlichen Hostnamens des Servers einen anderen Hostnamen für den DNS-MX- Eintrag einzugeben , z. B. secmx.yourdomain.com . Stellen Sie jedoch sicher, dass alles, was Sie eingeben, ein gültiger DNS-Name ist, der auf demselben Server aufgelöst wird.

Wenn Sie auf die Schaltfläche Speichern und hinzufügen klicken, werden alle vorhandenen virtuellen Server, für die E-Mail aktiviert ist, allen ausgewählten sekundären Mailservern hinzugefügt. Darüber hinaus werden DNS-MX-Einträge zu ihren Domänen hinzugefügt, um auf die Mailserver zu verweisen.
