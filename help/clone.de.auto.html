Virtuellen Server klonen
Auf dieser Seite können Sie einen vorhandenen virtuellen Server mit einem neuen Domänennamen duplizieren. Alle Webinhalte, DNS-Einträge, Postfächer, E-Mail-Aliase, Datenbanken und anderen Einstellungen, die der ursprünglichen Domain zugeordnet sind, werden nach Möglichkeit dupliziert. Der Webinhalt wird jedoch in keiner Weise geändert. Wenn der virtuelle Server eine Webanwendung enthält, die für die Verwendung der Datenbanken oder des Basisverzeichnisses des alten Servers konfiguriert wurde, muss dieser nach dem Klonen manuell neu konfiguriert werden.

Wenn Sie einen virtuellen Server der obersten Ebene duplizieren, müssen Sie auch einen neuen Benutzernamen und ein neues Kennwort eingeben, um die neue Domäne zu besitzen. Wenn der Server eine private IP-Adresse hat, wird dem Klon automatisch eine neue IP zugewiesen.

Beim Klonen werden alle mit der ursprünglichen Domäne verknüpften Datenbanken unter neuen Namen kopiert und deren Inhalt dupliziert. Es werden jedoch KEINE in der ursprünglichen Domäne installierten Skripts aktualisiert, die für die Verwendung der ursprünglichen Datenbanken konfiguriert sind. Dies muss manuell erfolgen.
