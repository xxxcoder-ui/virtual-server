Inclure les sous-chemins dans redirection?
Lorsque ce paramètre est réglé sur Oui, et sous-répertoire ou nom de fichier dans le chemin source d'URL ne seront pas inclus dans la redirection. Par exemple, si vous avez redirigé /google à http://www.google.com et un navigateur accède http://example.com/google/something , l'URL de destination serait encore http://www.google.com.

Si Non a été sélectionné dans cet exemple, la destination serait http://www.google.com/something plutôt.

Cependant, si Oui est choisi vous pouvez utiliser le code spécial $1 dans l'URL de destination. Ce sera remplacé par le sous-chemin ou le nom du fichier qui vient après le chemin source d'URL.
