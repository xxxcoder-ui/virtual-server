Serveurs de messagerie secondaires
Un serveur de messagerie secondaire est un serveur qui peut également recevoir des e-mails pour l'un des serveurs virtuels gérés par Virtualmin, mais uniquement comme relais temporaire plutôt que comme destination finale. Il est sage de configurer un serveur secondaire pour les domaines de messagerie très utilisés, afin de servir de lieu de sauvegarde ou de conservation en cas de panne du serveur de messagerie principal.

Sur cette page, vous pouvez sélectionner d'autres serveurs sur lesquels Virtualmin est installé pour être utilisés comme serveurs MX secondaires pour les nouveaux domaines de messagerie créés sur ce système. Avant que des serveurs n'apparaissent, ils doivent d'abord être ajoutés au module Webmin Servers Index, dans lequel vous devez entrer le login root et le mot de passe du serveur distant.

Pour chaque serveur de messagerie secondaire, vous avez la possibilité de saisir un nom d'hôte différent à utiliser dans l'enregistrement DNS MX, au lieu du nom d'hôte réel du serveur, tel que secmx.votredomaine.com . Cependant, assurez-vous que tout ce que vous entrez est un nom DNS valide qui se résout sur le même serveur.

Si vous cliquez sur le bouton Enregistrer et ajouter , tous les serveurs virtuels existants pour lesquels le courrier électronique est activé seront ajoutés à tous les serveurs de messagerie secondaires sélectionnés. De plus, les enregistrements DNS MX seront ajoutés à leurs domaines pour faire référence aux serveurs de messagerie.
