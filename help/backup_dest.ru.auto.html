Резервное копирование
Эти поля определяют, куда записывается резервная копия Virtualmin. Вы можете выбрать один из трех режимов назначения:
Локальный файловый каталог
В этом режиме резервная копия просто сохраняется в файл в системе, в которой работает Virtualmin. Если используется формат резервной копии файла единого архива , вы должны ввести здесь имя файла, например /backup/domainname.tgz . Если вы используете какой-либо файл по одному на серверные форматы, вам нужно вместо этого ввести каталог, например /backup .
FTP сервер
Этот режим можно использовать для загрузки файла резервной копии или файлов по протоколу FTP на другой сервер. Вы должны ввести имя сервера, логин, пароль и целевой файл или каталог.
SSH сервер
Это похоже на режим FTP, но вместо этого передает резервную копию с использованием протокола SSH.
Если формат резервной копии - один файл, то указанным здесь местом назначения должен быть файл, который будет перезаписан или создан. Если для каждого домена создается один файл, то местом назначения должен быть каталог, который уже существует.

Если вы выполняете резервное копирование на сервер SSH или FTP на нестандартном порту, вы можете ввести его в поле сервера, отделенное от имени хоста двоеточием, например backupserver.yourdomain.com:2222 .

При резервном копировании на сервер SSH, который поддерживает только аутентификацию на основе сертификатов, поле пароля является необязательным. Однако пользователь root в системе должен иметь свой открытый ключ SSH в файле authorized_keys конечного пользователя.
