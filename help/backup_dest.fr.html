Destination de sauvegarde
Ces champs contrôlent où la sauvegarde Virtualmin est écrit. Vous pouvez choisir à partir de l'un des trois modes de destination:
Répertoire de fichiers local
Dans ce mode, la sauvegarde est simplement enregistrée dans un fichier sur le système exécutant Virtualmin. Si l'on utilise le Seul fichier d'archive format de sauvegarde, vous devez entrer un nom de fichier ici, comme /backup/domainname.tgz. Si vous utilisez l'un des formats Un fichier par serveur, vous devez entrer un répertoire à la place, comme /backup.
Serveur FTP
Ce mode peut être utilisé pour avoir le fichier de sauvegarde ou des fichiers téléchargés à l'aide du protocole FTP à un autre serveur. Vous devez entrer le nom du serveur, login, mot de passe et fichier de destination ou répertoire.
Serveur SSH
Ceci est similaire au mode de FTP, mais transfère la sauvegarde à l'aide du protocole SSH.
Si le format de sauvegarde est un fichier unique, puis la destination que vous spécifiez ici doit être un fichier, qui sera écrasée ou créé. Si la création d'un fichier par domaine, la destination doit être un répertoire qui existe déjà.

Si vous sauvegarde sur un serveur FTP ou SSH sur un port non standard, vous pouvez entrer dans le champ de serveur séparé du nom d'hôte avec un côlon, comme backupserver.yourdomain.com:2222 .

Si vous sauvegarde sur un serveur SSH qui ne prend en charge l'authentification basée sur certifcate, le champ mot de passe est facultatif. Cependant, le root doit avoir sa clé publique SSH dans le fichier de destination authorized_keys.
