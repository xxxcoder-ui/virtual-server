Mode d'exection Ruby par défaut
Cette option détermine si les scripts Ruby des nouveaux serveurs virtuels sont exécutés en tant qu'utilisateur du serveur Web Apache (généralement httpd ou apache ) ou en tant qu'utilisateur Unix propriétaire du serveur virtuel. Le premier est plus rapide car il signifie qu'aucun processus supplémentaire ne doit être démarré pour exécuter le code Ruby, mais moins sécurisé lorsque les scripts lisent et écrivent des fichiers en tant qu'utilisateur autre que le propriétaire du domaine.
