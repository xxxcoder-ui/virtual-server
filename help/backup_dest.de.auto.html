Sicherungsziel
Diese Felder steuern, wohin die Virtualmin-Sicherung geschrieben wird. Sie können zwischen drei Zielmodi wählen:
Lokales Dateiverzeichnis
In diesem Modus wird die Sicherung einfach in einer Datei auf dem System gespeichert, auf dem Virtualmin ausgeführt wird. Wenn Sie das Sicherungsformat für einzelne Archivdateien verwenden, müssen Sie hier einen Dateinamen eingeben, /backup/domainname.tgz . B. /backup/domainname.tgz . Wenn Sie eines der Formate Eine Datei pro Server verwenden, müssen Sie stattdessen ein Verzeichnis wie /backup eingeben.
FTP-Server
In diesem Modus können die Sicherungsdatei oder Dateien mithilfe des FTP-Protokolls auf einen anderen Server hochgeladen werden. Sie müssen den Servernamen, das Login, das Passwort und die Zieldatei oder das Verzeichnis eingeben.
SSH-Server
Dies ähnelt dem FTP-Modus, überträgt die Sicherung jedoch stattdessen über das SSH-Protokoll.
Wenn das Sicherungsformat eine einzelne Datei ist, muss das hier angegebene Ziel eine Datei sein, die überschrieben oder erstellt wird. Wenn Sie eine Datei pro Domäne erstellen, muss das Ziel ein bereits vorhandenes Verzeichnis sein.

Wenn Sie auf einem SSH- oder FTP-Server an einem nicht standardmäßigen Port backupserver.yourdomain.com:2222 , können Sie ihn in das vom Hostnamen getrennte backupserver.yourdomain.com:2222 mit einem Doppelpunkt backupserver.yourdomain.com:2222 . B. backupserver.yourdomain.com:2222 .

Wenn Sie auf einem SSH-Server sichern, der nur die zertifikatbasierte Authentifizierung unterstützt, ist das Kennwortfeld optional. Der root Benutzer auf dem System muss jedoch seinen öffentlichen SSH-Schlüssel in der Datei authorized_keys Zielbenutzers haben.
