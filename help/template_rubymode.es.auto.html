Modo de ejecución de Ruby predeterminado
Esta opción determina si los scripts de Ruby en los nuevos servidores virtuales se ejecutan como el usuario del servidor web Apache (generalmente httpd o apache ), o el usuario Unix que posee el servidor virtual. El primero es más rápido, ya que significa que no es necesario iniciar procesos adicionales para ejecutar el código Ruby, pero menos seguro ya que los scripts leen y escriben archivos como un usuario que no sea el propietario del dominio.
