Destino de respaldo
Estos campos controlan dónde se escribe la copia de seguridad de Virtualmin. Puede elegir uno de los tres modos de destino:
Directorio de archivos locales
En este modo, la copia de seguridad simplemente se guarda en un archivo en el sistema que ejecuta Virtualmin. Si utiliza el formato de copia de seguridad de archivo de archivo único, debe ingresar un nombre de archivo aquí, como /backup/domainname.tgz . Si usa cualquiera de los formatos de Un archivo por servidor , debe ingresar un directorio, como /backup .
Servidor FTP
Este modo se puede usar para cargar el archivo de respaldo o los archivos usando el protocolo FTP a otro servidor. Debe ingresar el nombre del servidor, inicio de sesión, contraseña y el archivo o directorio de destino.
Servidor SSH
Esto es similar al modo FTP, pero transfiere la copia de seguridad utilizando el protocolo SSH.
Si el formato de copia de seguridad es un archivo único, el destino que especifique aquí debe ser un archivo, que se sobrescribirá o creará. Si crea un archivo por dominio, el destino debe ser un directorio que ya existe.

Si realiza una copia de seguridad en un servidor SSH o FTP en un puerto no estándar, puede ingresarlo en el campo del servidor separado del nombre de host con dos puntos, como backupserver.yourdomain.com:2222 .

Si realiza una copia de seguridad en un servidor SSH que solo admite autenticación basada en certifcados, el campo de contraseña es opcional. Sin embargo, el usuario root en el sistema debe tener su clave pública SSH en el archivo authorized_keys claves del usuario de destino.
