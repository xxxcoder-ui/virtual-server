Вторичные почтовые серверы
Вторичный почтовый сервер также может принимать электронную почту для одного из виртуальных серверов, управляемых Virtualmin, но только как временный ретранслятор, а не как конечный пункт назначения. Целесообразно настроить дополнительный сервер для интенсивно используемых почтовых доменов, который будет действовать в качестве резервного или временного хранилища в случае сбоя основного почтового сервера.

На этой странице вы можете выбрать другие серверы с установленным Virtualmin для использования в качестве вторичных серверов MX для новых почтовых доменов, созданных в этой системе. Прежде чем появятся какие-либо серверы, их необходимо сначала добавить в модуль индекса Webmin Servers, в котором необходимо ввести логин и пароль root для удаленного сервера.

Для каждого вторичного почтового сервера у вас есть возможность ввести другое имя хоста для использования в DNS-записи MX вместо фактического имени хоста сервера, например secmx.yourdomain.com . Однако убедитесь, что все, что вы вводите, является допустимым DNS-именем, которое разрешается на тот же сервер.

Если нажать кнопку «Сохранить и добавить» , все существующие виртуальные серверы, для которых включена электронная почта, будут добавлены на все выбранные вторичные почтовые серверы. Кроме того, записи DNS MX будут добавлены в их домены для ссылки на почтовые серверы.
